# Cross-Space Phenotyping of Polycystic Ovary Syndrome (PCOS)

## Overview

This repository contains the complete computational workflow used in the manuscript:

> **No single biological phenotype exists in polycystic ovary syndrome: evidence from cross-space phenotyping**

The repository provides a fully reproducible implementation of the cross-space phenotyping framework developed to investigate whether independently derived endocrine, metabolic, inflammatory, and thyroid phenotypes identify a common biological architecture or represent complementary dimensions of PCOS heterogeneity.

## Analytical workflow

The complete analysis is implemented as sequential notebooks and should be executed in numerical order.

The workflow includes:

1. Data import and harmonization
2. Quality control
3. Definition of biological spaces
4. Data preprocessing
5. Principal Component Analysis (PCA)
6. Gaussian Mixture Model (GMM) clustering
7. Bootstrap stability analysis
8. Cross-space agreement analysis
9. Biological characterization
10. Independent non-circular biological validation
11. Sensitivity analyses
12. Generation of publication-ready figures and tables

---

## Biological spaces

Phenotypes were identified independently within four predefined biological domains:

- Endocrine
- Metabolic
- Inflammatory
- Thyroid

Each biological space was analyzed using an identical computational pipeline.

## Data availability

The original clinical dataset is **not included** in this repository.

The study is based on anonymized patient data obtained during routine clinical care. Owing to ethical and privacy restrictions, these data cannot be made publicly available.

Researchers interested in reproducing the analytical workflow may adapt the notebooks to appropriately formatted datasets obtained in accordance with local ethical regulations.

## Reproducibility

The analytical pipeline was designed to maximize computational reproducibility.

- All analytical stages are implemented as sequential notebooks.
- Intermediate outputs generated by one notebook serve as inputs for subsequent analyses.
- Randomized procedures use fixed random seeds whenever applicable.
- The workflow reproduces all analyses reported in the manuscript.
